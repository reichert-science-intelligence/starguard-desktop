## 2. TYPOGRAPHY HIERARCHY

### Primary Font Family: **Montserrat** (Google Fonts / Canva Pro)

**Why:** Modern, clean, professional, excellent readability, healthcare-appropriate

### Secondary Font Family: **Open Sans** (Fallback)

**Why:** Web-safe, highly readable, professional alternative

### Typography Scale

#### H1 - Main Headlines
- **Font:** Montserrat Bold
- **Size:** 48-60pt (Desktop) / 36-42pt (Mobile)
- **Line Height:** 1.2
- **Color:** Navy Blue (#1B4965)
- **Usage:** Hero section, major section titles
- **Example:** "ROBERT REICHERT"

#### H2 - Section Headlines
- **Font:** Montserrat SemiBold
- **Size:** 32-40pt (Desktop) / 28-32pt (Mobile)
- **Line Height:** 1.3
- **Color:** Navy Blue (#1B4965)
- **Usage:** Section titles, feature boxes
- **Example:** "THE HEALTHCARE AI PARADOX"

#### H3 - Subsection Headlines
- **Font:** Montserrat Medium
- **Size:** 24-28pt (Desktop) / 20-24pt (Mobile)
- **Line Height:** 1.4
- **Color:** Navy Blue (#1B4965) or Charcoal (#2C3E50)
- **Usage:** Subsection titles, card titles
- **Example:** "SECURITY-FIRST"

#### Body Text - Primary
- **Font:** Open Sans Regular
- **Size:** 16-18pt (Desktop) / 14-16pt (Mobile)
- **Line Height:** 1.6
- **Color:** Charcoal (#2C3E50)
- **Usage:** Paragraphs, descriptions, general content

#### Body Text - Secondary
- **Font:** Open Sans Regular
- **Size:** 14-16pt (Desktop) / 12-14pt (Mobile)
- **Line Height:** 1.5
- **Color:** Charcoal (#2C3E50) at 80% opacity
- **Usage:** Supporting text, captions, metadata

#### Large Numbers (Metrics)
- **Font:** Montserrat Bold
- **Size:** 64-72pt (Desktop) / 48-56pt (Mobile)
- **Line Height:** 1.0
- **Color:** Teal (#62B6CB) or Healthcare Green (#00A86B)
- **Usage:** Key metrics, impact numbers
- **Example:** "$148M+"

#### Small Text (Labels, Captions)
- **Font:** Open Sans Regular
- **Size:** 12-14pt (Desktop) / 11-12pt (Mobile)
- **Line Height:** 1.4
- **Color:** Charcoal (#2C3E50) at 70% opacity
- **Usage:** Labels, captions, fine print

### Typography Rules

- **Maximum line length:** 65-75 characters for optimal readability
- **Headline spacing:** 1.5x font size below headlines
- **Paragraph spacing:** 1.2x font size between paragraphs
- **Never use:** Comic Sans, Papyrus, or decorative fonts
- **Emphasis:** Use **bold** for important words, *italic* sparingly
- **Numbers:** Always use Montserrat for large numbers/metrics

## 3. SPACING GUIDELINES

### Section Padding

- **Large Sections (Hero, Featured Project):** 80-100px top/bottom
- **Standard Sections:** 60-80px top/bottom
- **Compact Sections (Metrics, Timeline):** 40-60px top/bottom
- **Side Padding:** 40-60px (Desktop) / 20-30px (Mobile)

### Element Margins

- **Between Headlines and Body:** 24-32px
- **Between Paragraphs:** 16-24px
- **Between Cards/Boxes:** 24-32px
- **Between Icons and Text:** 12-16px
- **Between Buttons:** 16-24px

### White Space Principles

- **Rule of Thirds:** Divide sections into thirds for visual balance
- **Breathing Room:** Never let elements feel cramped - minimum 20px between interactive elements
- **Grouping:** Related elements should be closer (16-24px) than unrelated (40-60px)
- **Visual Hierarchy:** More space around important elements (headlines, CTAs)

### Grid System

- **Desktop:** 12-column grid with 20px gutters
- **Tablet:** 8-column grid with 16px gutters
- **Mobile:** 4-column grid with 12px gutters
- **Content Width:** Maximum 1200px for readability

## 4. ICON STYLE RECOMMENDATIONS

### Icon Style: **Line Icons** (Preferred)

**Why:** Modern, clean, professional, less visual weight

### Icon Specifications

- **Size:** 32-48px for standard icons, 64-80px for feature box icons
- **Stroke Width:** 2-3px for line icons
- **Color:** Teal (#62B6CB) for primary icons, Healthcare Green (#00A86B) for positive indicators
- **Style:** Rounded corners, consistent line weight
- **Library:** Canva's icon library, search "line icons" or "outline icons"

### Icon Usage by Context

- **Security/Lock:** Shield or lock icon in Teal (#62B6CB)
- **Metrics/Charts:** Bar chart or line graph in Teal (#62B6CB)
- **Code/Development:** Code brackets or terminal in Teal (#62B6CB)
- **Success/Checkmarks:** Checkmark in Healthcare Green (#00A86B)
- **Problems/Blocked:** X or warning in Warning Red (#E74C3C)
- **Contact/Social:** Use brand colors for LinkedIn, GitHub, Email icons

### Icon Consistency Rules

- **Same style throughout:** Don't mix filled and line icons
- **Same size within groups:** All feature box icons should be same size
- **Color coding:** Use color to indicate category (security = teal, success = green)
- **Alignment:** Icons should align with text baseline or center-align with text

### Alternative: Filled Icons (Use Sparingly)

- **When to use:** For emphasis, CTAs, or when line icons don't read well
- **Style:** Solid fill with 10-15% opacity background circle
- **Size:** Same as line icons (32-48px standard, 64-80px for features)

## 5. IMAGE TREATMENT

### Screenshot Specifications

- **Aspect Ratio:** 16:9 or 16:10 (standard widescreen)
- **Border:** 2-4px rounded corners (8-12px radius)
- **Shadow:** Subtle drop shadow (offset: 0px, blur: 8-12px, opacity: 20-30%)
- **Quality:** Minimum 1920px width for desktop, 1200px for mobile

### Headshot Specifications

- **Shape:** Circular crop (perfect circle)
- **Size:** 200-250px diameter (Desktop) / 150-180px (Mobile)
- **Border:** 4-6px white border with subtle shadow
- **Quality:** High resolution (minimum 400x400px source image)
- **Positioning:** Right side of hero section, aligned with name

### Background Images

- **Opacity Overlay:** 20-30% dark overlay (Navy Blue #1B4965 at 20% opacity)
- **Blur Effect:** Optional - 2-4px blur for subtle texture
- **Positioning:** Center or focal point alignment
- **Quality:** High resolution, professional stock photos

### Image Treatment Rules

- **Consistency:** All screenshots should have same border/shadow treatment
- **Compression:** Optimize for web (JPG for photos, PNG for screenshots with text)
- **Alt Text:** Always include descriptive alt text for accessibility
- **Never:** Stretch images, use low-resolution images, or pixelated graphics

### Image Placeholders

- **Screenshots:** Use actual Streamlit app screenshots
- **Architecture Diagrams:** Create in Canva using shapes and arrows
- **Backgrounds:** Use Canva's stock photos (search "healthcare technology" or "medical data")

## 6. BUTTON STYLES

### Primary CTA Button

**Style:**
- **Background:** Bright Teal (#4ECDC4)
- **Text:** White (#FFFFFF)
- **Font:** Montserrat SemiBold, 18-20pt
- **Padding:** 16px vertical, 32px horizontal
- **Border Radius:** 8-12px (rounded corners)
- **Shadow:** Subtle (offset: 0px, blur: 4-6px, opacity: 20%)
- **Hover Effect:** Slightly darker teal (#3DB5AD), slight elevation

**Usage:** Main call-to-action ("View Projects", "Schedule a Conversation")

**Example:**
```
[View Projects]
Background: #4ECDC4
Text: #FFFFFF
Size: 18pt Montserrat SemiBold
Padding: 16px × 32px
```

### Secondary Button

**Style:**
- **Background:** White (#FFFFFF) with 2px border
- **Border Color:** Teal (#62B6CB)
- **Text:** Teal (#62B6CB)
- **Font:** Montserrat Medium, 16-18pt
- **Padding:** 14px vertical, 28px horizontal
- **Border Radius:** 8-12px
- **Shadow:** None or very subtle

**Usage:** Secondary actions ("Download Resume", "View GitHub")

**Example:**
```
[Download Resume]
Background: #FFFFFF
Border: 2px #62B6CB
Text: #62B6CB
Size: 16pt Montserrat Medium
```

### Text Link Style

**Style:**
- **Text:** Teal (#62B6CB)
- **Font:** Open Sans Regular, 16pt
- **Underline:** On hover only (optional)
- **Decoration:** None by default

**Usage:** Inline links, social media links

### Button Spacing

- **Between buttons:** 16-24px horizontal spacing
- **Button groups:** Center-align or left-align based on context
- **Touch targets:** Minimum 44px height for mobile accessibility

### Button Rules

- **Maximum 2-3 buttons per section** (avoid choice paralysis)
- **Clear hierarchy:** Primary button should be most prominent
- **Consistent sizing:** All primary buttons same size, all secondary buttons same size
- **Accessibility:** High contrast (4.5:1 minimum for text on background)

## 7. MOBILE RESPONSIVENESS NOTES

### Layout Changes

**Desktop (1200px+):**
- Multi-column layouts (2-4 columns)
- Side-by-side content
- Horizontal timelines
- Icon grids

**Tablet (768px - 1199px):**
- 2-column layouts become single column
- Stack feature boxes vertically
- Reduce padding by 20%
- Maintain horizontal timelines (if space allows)

**Mobile (< 768px):**
- Single column for all content
- Stack all elements vertically
- Full-width buttons
- Simplified navigation
- Reduced padding (40-60px → 20-30px)

### Text Size Adjustments

**Desktop → Mobile:**
- H1: 60pt → 36pt (40% reduction)
- H2: 40pt → 28pt (30% reduction)
- H3: 28pt → 20pt (29% reduction)
- Body: 18pt → 16pt (11% reduction)
- Large Numbers: 72pt → 48pt (33% reduction)

### Touch-Friendly Elements

- **Button Height:** Minimum 44px (Apple HIG recommendation)
- **Button Spacing:** Minimum 16px between buttons
- **Icon Size:** Minimum 32px for touch targets
- **Link Spacing:** Minimum 8px padding around clickable areas

### Mobile-Specific Optimizations

- **Hero Section:** Stack headshot above name (not side-by-side)
- **Feature Boxes:** Full-width, stacked vertically
- **Timeline:** Vertical timeline instead of horizontal
- **Screenshots:** Full-width, maintain aspect ratio
- **Buttons:** Full-width or centered, larger touch targets
- **Navigation:** Hamburger menu or simplified navigation

### Mobile Testing Checklist

- [ ] All text readable without zooming
- [ ] All buttons easily tappable (44px+ height)
- [ ] Images load quickly (optimized file sizes)
- [ ] No horizontal scrolling
- [ ] All links work on mobile
- [ ] Forms/inputs are mobile-friendly
- [ ] Social media icons are tappable

## 11. ACCESSIBILITY CHECKLIST

- [ ] Color contrast meets WCAG AA standards (4.5:1 for text)
- [ ] All text is readable (minimum 12pt, preferably 14pt+)
- [ ] Icons have text labels or are decorative only
- [ ] Buttons have clear labels and are large enough (44px+)
- [ ] Images have alt text descriptions
- [ ] Navigation is keyboard accessible
- [ ] Focus states are visible for interactive elements
- [ ] No color-only information (use icons or text too)

---

## 12. FINAL DESIGN CHECKLIST

Before publishing, verify:

- [ ] All colors match the palette
- [ ] Typography is consistent throughout
- [ ] Spacing is consistent (use Canva's alignment guides)
- [ ] Icons are same style and size within groups
- [ ] Buttons follow the style guide
- [ ] Images are high quality and properly sized
- [ ] Mobile version is tested and optimized
- [ ] All links work and point to correct destinations
- [ ] Brand colors are saved in Canva Brand Kit
- [ ] Export settings are optimized for intended use

---

**Remember:** Consistency is key. Once you establish these design rules, apply them consistently across all sections. This creates a professional, cohesive brand identity that builds trust with recruiters and hiring managers.
