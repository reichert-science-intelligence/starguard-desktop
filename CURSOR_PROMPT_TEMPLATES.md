Use these prompts in Cursor to generate customized content quickly.

## Cover Letter Generation

### Template 1: Standard Application

```
Generate a cover letter for [COMPANY] [POSITION] emphasizing:

1. My secure healthcare AI expertise (HIPAA-compliant LLM deployment)

2. Specific project: [CHOOSE: HEDIS Portfolio Optimizer OR Glycemic Screening Model OR Healthcare AI Chatbot]

3. How I solve their pain point: [EXTRACT FROM JOB DESCRIPTION - e.g., "need AI capabilities without PHI exposure"]

4. Quantified impact: [CHOOSE ONE: $148M savings OR 93% recall OR 2.8-4.1x ROI OR $380K annual value]

5. Close with: My approach bridges the gap between AI innovation and healthcare compliance that [COMPANY] needs for [SPECIFIC INITIATIVE FROM JOB DESCRIPTION]

Keep to 3 paragraphs, 250 words max. Professional but conversational tone.
```

### Template 2: Senior/Lead Position

```
Generate a senior-level cover letter for [COMPANY] [POSITION] that positions me as a thought leader in secure healthcare AI:

1. Open with: My track record solving the $50B healthcare AI adoption barrier through on-premises deployment architecture

2. Highlight: [SPECIFIC PROJECT] demonstrating [QUANTIFIED RESULT] while maintaining zero external API exposure

3. Connect to: Their need for [PAIN POINT FROM JOB DESCRIPTION] and how secure AI architecture solves it

4. Close: Offer to lead [COMPANY]'s secure AI strategy, bridging innovation and compliance

Tone: Confident, strategic, results-focused. 300 words max.
```

## Resume Bullet Generation

### Template 1: Single Bullet

```
Transform this resume bullet to emphasize security and impact:

Original: "[YOUR CURRENT BULLET POINT]"

Transform using this format:
"[Architected/Engineered] HIPAA-compliant [TECHNOLOGY] [ACHIEVEMENT] while [SECURITY FEATURE], [QUANTIFIED IMPACT]"

Include: Security compliance, technology specifics, quantified business value.
```

### Template 2: Multiple Bullets for One Project

```
Generate 3 resume bullets for [PROJECT NAME] emphasizing:

1. First bullet: Security architecture and compliance
2. Second bullet: Technical implementation and scale
3. Third bullet: Business impact and quantified results

Each bullet should:
- Start with strong action verb
- Mention HIPAA/security
- Include technology specifics
- Quantify impact (numbers, percentages, dollars)

Keep each bullet to 2 lines max.
```

## Project Documentation

### Template 1: Project Summary

```
Generate a project summary for [PROJECT NAME] following this structure:

1. Overview (2-3 sentences): What it does, security approach
2. Security Architecture (bullet points): On-premises processing, encryption, zero exposure
3. Technical Stack (bullet points): Technologies used
4. Impact (quantified): ROI, efficiency, cost savings
5. Compliance: HIPAA considerations, audit capabilities

Keep each section concise. Professional technical writing.
```

### Template 2: Security Architecture Description

```
Write a security architecture description for [PROJECT NAME]:

1. Data flow: User → Local Processing → Database → Response
2. Security controls: Encryption (rest/transit), access controls, audit logging
3. Compliance: HIPAA measures, PHI protection, audit trail
4. Zero exposure guarantee: No external API calls, complete data sovereignty

Technical but accessible. Include diagram description if needed.
```

## Customization Instructions

### How to Use These Templates:

1. **Copy the template** you need
2. **Replace bracketed placeholders** with actual information:
   - [COMPANY] → Actual company name
   - [POSITION] → Job title
   - [PROJECT NAME] → Your specific project
   - [QUANTIFIED IMPACT] → Your actual metrics
3. **Paste into Cursor** with the template
4. **Review and refine** the generated content
5. **Customize further** based on specific job/context

## Tips for Best Results

1. **Be Specific**: The more context you provide, the better the output
2. **Include Job Description**: Copy relevant parts for better customization
3. **Add Company Research**: Mention specific initiatives/projects if known
4. **Specify Tone**: Professional, conversational, technical, etc.
5. **Set Word Limits**: Keep content concise and focused
6. **Review Always**: AI-generated content needs human review and refinement
